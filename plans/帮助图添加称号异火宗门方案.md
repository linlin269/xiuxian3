# 帮助图添加称号、异火、宗门三个分组方案

## 目标

在 [`生成修仙帮助图.py`](../xiuxian3/xiuxianserver/test/生成修仙帮助图.py) 的 `SECTIONS` 列表中新增三个分组卡片，分别对应称号、异火和宗门模块的 WS 命令。

## 现有格式分析

每个分组是一个 dict，包含三个字段：

```python
{
    "title": "分组标题",
    "color": "#RRGGBB",  # 卡片头部背景色
    "lines": ["命令1", "命令2", ...],  # 每行一条命令
}
```

- `title`：卡片标题，中文短语
- `color`：十六进制颜色值，暖色/粉彩风格
- `lines`：命令列表，每条命令就是一个字符串；带参数的命令用空格分隔展示

## 插入位置

建议在 `SECTIONS` 列表中，将三个新分组插入在「首领」之后、「修仙界历史」之前：

```text
... 首领 → 称号 → 异火 → 宗门 → 修仙界历史
```

理由：
- 修仙界历史是总结性的信息模块，放在最后作为收尾
- 称号、异火、宗门是功能玩法模块，放在首领（战斗类末尾）之后逻辑连贯

## 修改点

仅修改一个文件：[`xiuxian3/xiuxianserver/test/生成修仙帮助图.py`](../xiuxian3/xiuxianserver/test/生成修仙帮助图.py)

在 `SECTIONS` 列表的第 191 行（首领分组结束 `},` 之后），第 192 行（修仙界历史分组 `{` 之前），插入三个新 dict。

## 布局影响

- 当前 14 个分组 → 新增 3 个 → 共 17 个分组
- 3 列瀑布流布局，17 个卡片会均匀分配到 3 列
- 图片总高度会略微增加（约 3 张卡片的高度）
- 不需要修改布局逻辑，`main()` 中的瀑布流算法自动处理
