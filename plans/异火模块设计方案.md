# 异火模块设计方案

## 一、已确认边界

- 异火独立成体系，不复用纳戒物品、背包物品或武器实例。
- 异火提供独立入口：异火、异火列表、异火详情、异火装备、异火卸下、异火合成、异火交易。
- 异火定义来自异火列表.csv，共 23 种，排名、名称、加成、外观和效果固定落库。
- 第 1 名帝炎不掉落，只能由第 2 至第 23 名共 22 种异火合成。
- 第 21 至第 23 名异火可在任意区域探险里获得。
- 第 2 至第 23 名异火可在首领奖励或虫洞奖励里掉落。
- 异火奖励是最珍贵战利品；掉落权重从第 23 名到第 2 名逐级递减。
- 帝炎合成前，同一玩家每一种异火最多持有 1 个。
- 帝炎合成后，玩家不能再获得第 2 至第 23 名异火。
- 异火可玩家间交易、可上架二手市场，但禁止商城出售和普通回收。
- 异火装备占独立异火槽位，不挤占现有七件固定装备位。
- 异火装备后，人物最终攻击乘以该异火加成倍率，面板和战斗展示都显示乘算后的攻击。
- 掉落补偿固定文案：本次掉落异火为XXX，您已拥有此异火，本次奖励补偿为YYY。
- 合成失败固定文案：帝炎合成失败，缺少异火：A、B、C。

## 二、数据库表结构草案

### 1. 异火定义表

表名：flame_defs

| 字段 | 类型 | 说明 |
|---|---|---|
| flame_id | TEXT PRIMARY KEY | 稳定 ID，例如 di_yan |
| rank | INTEGER UNIQUE NOT NULL | 排名，1 至 23 |
| name | TEXT UNIQUE NOT NULL | 异火名称 |
| attack_multiplier | REAL NOT NULL | 攻击倍率，对应 csv 的加成 |
| appearance | TEXT NOT NULL | 外观描述 |
| effect_desc | TEXT NOT NULL | 效果描述 |
| source_type | TEXT NOT NULL | fusion、boss_wormhole、explore_low |
| tradeable | INTEGER NOT NULL DEFAULT 1 | 是否允许玩家间交易 |
| market_sellable | INTEGER NOT NULL DEFAULT 0 | 是否允许商城出售，固定为 0 |
| created_at | TEXT NOT NULL DEFAULT '' | 初始化或补种时间 |

落点建议：放在修仙根数据库初始化的定义表区域，和 item_defs、ring_item_defs、weapon_defs 同级，但不加入背包或纳戒定义。

### 2. 玩家异火持有表

表名：player_flames

| 字段 | 类型 | 说明 |
|---|---|---|
| client_id | TEXT NOT NULL | 玩家 ID |
| flame_id | TEXT NOT NULL | 异火 ID |
| equipped | INTEGER NOT NULL DEFAULT 0 | 是否装备 |
| source | TEXT NOT NULL DEFAULT '' | 来源：探险、首领、虫洞、合成、交易等 |
| obtained_at | TEXT NOT NULL | 获取时间 |
| updated_at | TEXT NOT NULL | 更新时间 |
| PRIMARY KEY | client_id, flame_id | 保证同一玩家同名异火最多 1 个 |

服务层约束：

- 插入前检查玩家是否已有帝炎。
- 如果玩家已有帝炎，拒绝写入 rank 2 至 rank 23 的异火，转入补偿逻辑。
- 如果玩家已有同名异火，拒绝重复插入，转入补偿逻辑。
- 装备异火时，同一玩家先把其它 equipped 置 0，再把目标置 1。

### 3. 异火合成记录表

表名：flame_fusion_records

| 字段 | 类型 | 说明 |
|---|---|---|
| record_id | INTEGER PRIMARY KEY AUTOINCREMENT | 记录 ID |
| client_id | TEXT NOT NULL | 玩家 ID |
| target_flame_id | TEXT NOT NULL | 固定为帝炎 |
| consumed_flames | TEXT NOT NULL | JSON 数组，成功时记录消耗的 22 种异火 |
| result | TEXT NOT NULL | success 或 failed |
| missing_flames | TEXT NOT NULL DEFAULT '[]' | 失败时缺失异火 JSON 数组 |
| created_at | TEXT NOT NULL | 创建时间 |

合成规则：

- 异火合成只处理帝炎。
- 检查 rank 2 至 rank 23 的 22 种异火。
- 缺失任意一种时不消耗已有异火，写入 failed 记录可选，返回固定失败文案。
- 全部满足时删除 rank 2 至 rank 23 的持有记录，插入帝炎，默认装备帝炎。
- 成功后写入 success 记录，并记录 consumed_flames。

### 4. 二手市场适配

现有 second_hand_listings 和 second_hand_records 的 item_type 是 TEXT，可新增 item_type = flame，不需要新建市场表。

二手市场规则：

- item_type 固定为 flame。
- item_id 保存 flame_id。
- quantity 固定为 1。
- 上架已装备异火时直接拒绝，提示先异火卸下。
- 上架时从 player_flames 删除该异火，避免卖家仍然持有效果。
- 下架时重新插入 player_flames，source 记为 二手市场下架。
- 购买时插入买家的 player_flames，source 记为 二手市场购买。
- 如果买家已有同名异火或已有帝炎导致不能接收，则购买失败，不转移资产。
- 手续费、成交记录和通知继续复用二手市场现有逻辑。
- 商城出售、自动出售、出售全部等贸易服务入口不处理 flame。

## 三、命令入口边界

建议新增二级组件：修仙/异火/

文件结构：

- __init__.py：挂载 WS 命令。
- service.py：异火业务服务。
- 说明.md：异火组件文档。

建议命令：

| 命令 | 作用 |
|---|---|
| 异火 | 查看自己持有异火、当前装备异火和入口按钮 |
| 异火列表 | 查看 23 种异火列表、排名和倍率 |
| 异火详情 异火名 | 查看单个异火外观、效果、来源和倍率 |
| 异火装备 异火名 | 装备已拥有异火 |
| 异火卸下 | 卸下当前异火，攻击倍率回到 1.0 |
| 异火合成 | 尝试合成帝炎 |
| 异火交易 | 展示异火交易说明，提示使用二手市场上架和购买 |

不建议新增直接点对点转让命令。异火交易应复用二手市场：

- 上架：二手市场上架 异火名 1 总价
- 购买：二手市场购买 卖家名称

异火交易命令只作为说明页，不直接转移资产，避免绕过二手市场手续费、成交记录和通知。

## 四、掉落补偿规则细化

### 1. 统一掉落判定顺序

所有异火掉落入口统一调用异火服务，不在探险、首领、虫洞各自重复实现唯一性判断。

流程：

1. 掉落入口生成候选异火。
2. 异火服务读取候选异火定义和玩家持有状态。
3. 如果玩家已有帝炎且候选异火 rank 为 2 至 23，转补偿。
4. 如果玩家未有帝炎但已有同名候选异火，转补偿。
5. 如果可以获得，写入 player_flames，并返回获得异火文本。
6. 如果转补偿，从低一档奖励池抽取补偿，并返回固定补偿文案。

### 2. 补偿池定义

“比异火奖励低一个等级里的战利品”按入口回退到对应原有奖励池。

探险补偿：

- 回退到当前探险可产出的普通战利品或怪物战利品。
- 不再触发异火。
- 如果背包无法容纳，补偿文本保留，并追加无法入包原因。

首领补偿：

- 回退到首领原本珍贵战利品池，但排除异火。
- 可补偿开孔器、洗髓液、宝石、技能书、武器、铭刻之羽等现有首领奖励。
- 同一次首领奖励中，铭刻之羽仍遵守最多 1 枚的现有约束。

虫洞补偿：

- 回退到虫洞原本珍贵奖励池，但排除异火。
- 可补偿洗髓液、宝石、技能书、武器、战备战利品等现有虫洞奖励。

补偿文本固定模板：

- 本次掉落异火为玄黄炎，您已拥有此异火，本次奖励补偿为纳戒获得 洗髓液 x1。
- 本次掉落异火为玄黄炎，您已拥有此异火，本次奖励补偿为宝石获得 护心玉 1级 x1。
- 本次掉落异火为玄黄炎，您已拥有此异火，本次奖励补偿为获得武器〔12〕青云剑[珍品] 上限60。

说明：帝炎合成后再触发 rank 2 至 23 的异火奖励，也按同一模板输出，不单独改文案。

## 五、异火掉落权重

### 1. 探险低阶异火

范围：rank 21、rank 22、rank 23。

建议规则：

- 任意普通区域探险均可触发。
- 太虚秘境是否算任意区域需要实现时再确认；默认纳入探险入口，但可配置排除。
- 候选权重按 rank 23 最高、rank 22 次之、rank 21 最低。
- 建议只在探险结算阶段增加一次极低概率判定，而不是每场怪物战斗都判定，避免概率膨胀。

### 2. 首领和虫洞异火

范围：rank 2 至 rank 23。

建议规则：

- 在原有珍贵战利品抽取中新增异火作为最高层级奖励。
- 异火命中后按 rank 23 到 rank 2 的权重池抽候选。
- 权重递减可以用线性或指数递减；建议指数递减，rank 2 极稀有。
- rank 1 帝炎不进入任何掉落池。

建议权重方向：

- rank 23 权重最大。
- rank 22 至 rank 10 逐级下降。
- rank 9 至 rank 3 明显稀有。
- rank 2 权重最低。

## 六、合成失败文案

固定失败文案：

- 帝炎合成失败，缺少异火：虚无吞炎、净莲妖火、玄黄炎。

边界：

- 如果缺少全部 22 种，就列出全部缺失名称。
- 如果已经拥有帝炎，再次异火合成返回：你已拥有帝炎，无需重复合成。
- 如果玩家没有创建用户，沿用修仙现有未建档提示。
- 合成失败不消耗已有异火。
- 合成成功后返回成功文本，并建议查看异火或修仙信息。

## 七、攻击倍率接入点

当前玩家面板与战斗多处使用基础攻击加武器攻击。实现时建议新增公共方法：

- 读取已装备异火倍率，默认 1.0。
- 计算最终攻击：整数化后攻击 = floor((base_attack + weapon_attack) * flame_multiplier)。
- 修仙信息、状态、战斗核心、首领/虫洞生态估算都调用统一最终攻击方法。

展示建议：

- 面板显示：攻击 1234，异火：玄黄炎 x1.100。
- 未装备异火时不强行展示，或展示异火：未装备。
- 战斗日志参数里记录最终攻击，避免网页和消息摘要不一致。

- 接入虫洞奖励，增加 rank 2 至 rank 23 的异火产出和补偿。
- 扩展二手市场支持 item_type = flame。

- 补充测试或冒烟覆盖：异火列表、装备、合成失败、重复掉落补偿、帝炎合成后锁定、二手市场上架购买、攻击倍率。
