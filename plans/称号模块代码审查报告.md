# 称号模块代码审查报告

## 错误概述

运行时出现 `sqlite3.OperationalError: no such column: amount`，调用链为：

```
玩家/ws_status → 玩家/service.status → common.refresh_titles
→ 称号/service.refresh_titles → refresh_titles_conn → _title_stats_conn
→ _sect_contribution → conn.execute SUM(amount)
```

## 数据库实际表结构

`sect_contribution_records` 表定义（[sql.py:1172-1184](xiuxian3/xiuxianserver/修仙/sql.py:1172)）：

| 列名 | 类型 | 说明 |
|------|------|------|
| record_id | INTEGER PK | 自增主键 |
| sect_id | INTEGER | 宗门ID |
| client_id | TEXT | 玩家ID |
| **influence** | INTEGER | 个人影响力 |
| **item_value** | INTEGER | 物资价值 |
| **success** | INTEGER | 成功次数 |
| cycle_start | TEXT | 周期开始 |
| cycle_end | TEXT | 周期结束 |
| detail | TEXT | 详情 |
| created_at | TEXT | 创建时间 |

**注意：表中没有 `amount` 列。**

## 发现的 Bug

### Bug-1（致命）：`称号/service.py` 第643-651行 `_sect_contribution`

```python
# 当前错误代码
row = conn.execute(
    """
    SELECT COALESCE(SUM(amount), 0) AS total    -- ❌ amount 列不存在
    FROM sect_contribution_records
    WHERE client_id = ?
    """,
    (client_id,),
).fetchone()
```

**修复方案**：将 `SUM(amount)` 改为 `SUM(influence)`。`influence` 是宗门个人贡献的核心度量值（参见 [sect_war.py:534-542](xiuxian3/xiuxianserver/修仙/sect_war.py:534) 的 INSERT 逻辑）。

### Bug-2（致命）：`common.py` 第1727-1730行 `_sect_contribution_conn`

```python
# 当前错误代码（与 Bug-1 完全相同的问题）
row = conn.execute(
    "SELECT COALESCE(SUM(amount), 0) AS total FROM sect_contribution_records WHERE client_id = ?",
    (client_id,),
).fetchone()
```

**注意**：此方法目前是死代码（`common.py` 的 `refresh_titles` 已委托给 `称号/service.py`），但仍应修复以保持一致性。

### Bug-3（潜在）：`common.py` 第1718-1724行 `_all_equipment_slots_opened`

```python
# 当前错误代码
rows = conn.execute(
    "SELECT holes_opened FROM equipment WHERE client_id = ?",  -- ❌ 表名和列名都错
    (client_id,),
).fetchall()
```

- 引用了不存在的 `equipment` 表（实际表名为 `fixed_equipment`）
- 引用了不存在的 `holes_opened` 列（实际列名为 `hole_count`）

对比 `称号/service.py` 第628-637行的正确实现：

```python
rows = conn.execute(
    "SELECT slot, hole_count FROM fixed_equipment WHERE client_id = ?",
    (client_id,),
).fetchall()
```

**注意**：此方法也是死代码（`_title_stats_conn` 已委托给 `称号/service.py`），但仍应修复。
