## 二、严重 Bug（会导致核心功能失效）

### Bug 1：十阶满星进阶成功后坐骑数据损坏，显化选择无法触发

**文件**：[`service.py` `_do_advance()`](xiuxian3/xiuxianserver/修仙/坐骑/service.py:623) 第 658-668 行

**问题**：当十阶「玉麒麟」满星进阶成功时，代码将 `mount_id` 更新为 `mount_def["post_advance_mount_id"]`，而 `qj_10` 的 `post_advance_mount_id` 为空字符串 `""`。进阶成功后：

```python
# service.py 第 658-668 行
next_id = mount_def["post_advance_mount_id"]  # qj_10 的值为 ""
self.db.execute(
    """
    UPDATE player_mounts
    SET mount_id=?, stars=0, blessing_value=0, blessing_expires_at='',
        manifest_chosen=0, updated_at=?
    WHERE client_id=?
    """,
    (next_id, ts(), client_id),  # mount_id 被设为空字符串!
)
```

结果：
1. `player_mounts.mount_id` 变为 `""`（空字符串）
2. `player_mounts.stars` 归零为 `0`
3. 随后调用 `_check_manifest_prompt` 时，`_get_mount_def("")` 返回 `None`，函数直接返回 `None`
4. **显化方向选择界面永远不会弹出**，玩家的坐骑数据彻底损坏

**影响**：所有到达十阶满星进阶的玩家都会遭遇此 bug，坐骑系统完全失效。

**修复方案**：在 `_do_advance` 中对十阶进阶成功做特殊处理——保持 `mount_id` 和 `stars` 不变，只清除祝福值并设 `manifest_chosen=0`：

```python
if _random.random() < prob:
    next_id = mount_def["post_advance_mount_id"]
    if next_id:
        # 普通进阶/显化进阶：更新为新坐骑
        self.db.execute(
            """UPDATE player_mounts
               SET mount_id=?, stars=0, blessing_value=0, blessing_expires_at='',
                   manifest_chosen=0, updated_at=?
               WHERE client_id=?""",
            (next_id, ts(), client_id),
        )
    else:
        # 十阶满星进阶成功：保持当前坐骑，等待显化选择
        self.db.execute(
            """UPDATE player_mounts
               SET blessing_value=0, blessing_expires_at='',
                   manifest_chosen=0, updated_at=?
               WHERE client_id=?""",
            (ts(), client_id),
        )
    success = True
    break
```

### Bug 2：`advance()` 和 `confirm_advance()` 参数解析错误，进阶功能完全不可用

**文件**：[`service.py`](xiuxian3/xiuxianserver/修仙/坐骑/service.py)

**问题**：框架 [`WsMessageHandler._split_message()`](xiuxian3/xiuxianserver/launch/adapter/ws/handler.py:216) 会在第一个空格处拆分消息。用户输入 `"坐骑进阶 10"` 会被拆分为 `cmd="坐骑进阶"`、`message="10"`。但代码用 `parts[1]` 取数量：

**`advance()` 第 437-440 行**：
```python
parts = message.strip().split()   # message 已是 "10"，split 后 parts = ["10"]
if len(parts) < 2 or not parts[1].isdigit():  # len=1 < 2，条件为 True
    return T.hint("格式：坐骑进阶 数量", ...)  # 永远返回格式错误提示
count = int(parts[1])  # 永远到不了这里
```

**`confirm_advance()` 第 581-584 行**：
```python
parts = message.strip().split()   # message 已是 "10"，split 后 parts = ["10"]
if len(parts) < 2 or not parts[1].isdigit():  # 同样永远为 True
    return T.hint("格式：确认进阶 数量。", ...)
```

**影响**：手动输入 `坐骑进阶 数量` 和点击确认进阶按钮均会返回格式错误提示，进阶功能完全无法使用。

**修复方案**：两个方法中将 `parts[1]` 改为 `parts[0]`，将 `len(parts) < 2` 改为 `len(parts) < 1` 或 `not parts`：

```python
# advance() 和 confirm_advance() 中均需修改
parts = message.strip().split()
if not parts or not parts[0].isdigit():
    return T.hint(...)
count = int(parts[0])
```

## 五、优化建议（非必须修复）

### 建议 7：`confirm_advance()` / `confirm_advance_all()` 缺少祝福值过期检查

**文件**：[`service.py`](xiuxian3/xiuxianserver/修仙/坐骑/service.py) 第 560 行、第 596 行

从二次确认展示到玩家点击确认之间，可能存在时间差。如果恰好跨越 24 小时边界，祝福值已过期但仍会使用过期数据执行进阶。建议在确认方法开头增加 `_check_blessing_expired(mount)` 调用。

### 建议 8：返回文本中按钮标记缺少换行分隔

**文件**：[`service.py`](xiuxian3/xiuxianserver/修仙/坐骑/service.py) 第 327-332 行、第 387-391 行、第 703-708 行、第 711-716 行

多处返回文本的最后一个 `<坐骑>` 标记紧跟上一行文本末尾，没有 `\n` 分隔。虽然 `split_button_tags` 能正确解析，但建议加换行保持格式一致性：

```python
# 当前（无换行）
f"消耗 {self._item_name(star_item)} x1"
f"<坐骑>"

# 建议（加换行）
f"消耗 {self._item_name(star_item)} x1\n"
f"<坐骑>"
```

### 建议 9：极显化坐骑 `star_up` / `star_up_all` 满星提示不当

**文件**：[`service.py`](xiuxian3/xiuxianserver/修仙/坐骑/service.py) 第 306-309 行

极显化坐骑满星时的提示：
```python
"可以尝试进阶提升品阶。<坐骑进阶>" if mount_def["mount_type"] != "extreme" else "<坐骑>"
```

极显化坐骑无法进阶，满星后应改为类似「极显化坐骑已满星，恭喜！」的提示。
