# 坐骑模块设计方案

## 一、概述

坐骑是玩家角色养成的独立子系统。玩家等级达到 10 级后自动开启坐骑模块，获得 0 星一阶坐骑「青角牛」。坐骑拥有品阶和星级两条成长线：升星消耗升星物品，满星后可使用进阶物品尝试进阶；进阶成功后品阶提升，星级归零。十阶满星进阶成功后进入显化分支，四选一获得显化坐骑；显化坐骑进阶后变为极显化坐骑，极显化坐骑只能升星不能进阶。

核心机制：

- 坐骑独立于背包、纳戒、武器和异火，不占用装备栏位。
- 坐骑信息展示在修仙信息面板中，包含品阶、星级、坐骑名称和意境说明。
- 升星和进阶消耗的物品纳入纳戒物品体系。
- 进阶概率公式统一为：`P = (当前进阶祝福值 / 进阶祝福总值) ^ 3`。
- 进阶祝福值有 24 小时有效期，从使用第一个进阶物品算起，到期未进阶则清空。
- 升星支持一键升星，进阶支持批量进阶和一键进阶，均提供便捷操作。

## 四、进阶祝福值机制详解

### 4.1 规则

1. 每次使用进阶物品（兽魂丹 / 幻兽魂丹），在该品阶规定的祝福值范围内随机获得一个整数祝福值。
2. 第一次使用进阶物品时开始计时，进阶祝福值有效期为 24 小时。
3. 24 小时内进阶成功，进阶祝福值清零，计时器清除。
4. 24 小时到期未进阶，进阶祝福值清零，计时器清除。
5. 进阶祝福值不可跨品阶累积；进阶失败（未成功判定）后继续累积直到成功或过期。

### 4.2 进阶概率公式

```
P = (当前进阶祝福值 / 进阶祝福总值) ^ 3
```

- 当进阶祝福值 = 0 时，P = 0，即不使用进阶物品时无法进阶。
- 当进阶祝福值 = 进阶祝福总值时，P = 1，即 100% 进阶成功。
- 使用进阶物品后立即进行一次进阶判定。

### 4.3 各品阶祝福值总览

| 品阶/阶段 | 进阶祝福总值 | 每次祝福值范围 | 理论最少使用次数 | 理论最多使用次数 |

| 1阶→2阶 | 4 | 1-1 | 4 | 4 |
| 2阶→3阶 | 16 | 1-2 | 8 | 16 |
| 3阶→4阶 | 64 | 1-3 | 22 | 64 |
| 4阶→5阶 | 256 | 1-4 | 64 | 256 |
| 5阶→6阶 | 1024 | 1-5 | 205 | 1024 |
| 6阶→7阶 | 4096 | 1-6 | 683 | 4096 |
| 7阶→8阶 | 16384 | 1-7 | 2341 | 16384 |
| 8阶→9阶 | 65536 | 1-8 | 8192 | 65536 |
| 9阶→10阶 | 262144 | 1-9 | 29127 | 262144 |

#### 坐骑升星

- 手动使用 1 个升星物品，提升 1 星。
- 升星物品：一阶至十阶为「兽血」，显化为「幻兽血」，极显化为「极幻兽血」。
- 满星时提示已满星，需进阶后才能继续升星。
- 缺少升星物品时提示物品不足。

**一键升星按钮：**

- 消耗玩家已拥有的升星物品，每消耗 1 个提升 1 星，直至满星。
- 达到满星后自动停止，不再消耗升星物品。
- 如果升星物品不足，则消耗完全部已有物品后停止，提示实际升星数和剩余星数。
- 如果当前已满星，点击后提示「当前坐骑已满星，无法继续升星」。
- 输出示例：

```text
一键升星完成！
消耗 兽血 x5
坐骑：霜月鹿 ⭐2/10 → ⭐7/10
[坐骑] [坐骑进阶] [一键进阶] [坐骑帮助]
```

#### 坐骑进阶 数量

- 格式：`坐骑进阶 数量`，例如 `坐骑进阶 10` 表示连续使用 10 个进阶物品。
- 前置条件：当前坐骑已满星。
- 数量为正整数，且不超过玩家拥有的进阶物品数量。
- **二次确认**：输入命令后，系统先展示确认信息，包含：
  - 当前坐骑名称和品阶
  - 当前进阶祝福值和进阶祝福总值
  - 将消耗的进阶物品名称和数量
  - 当前进阶概率和使用后的预计概率
  - 确认按钮和取消按钮
- 确认后逐个消耗进阶物品，每个消耗后进行一次进阶判定。
- 如果中途进阶成功，停止消耗剩余物品，提示进阶成功。
- 如果全部消耗完毕仍未进阶，提示当前祝福值和剩余状态。

**一键进阶按钮：**

- 消耗玩家已拥有的全部进阶物品，直至成功进阶或物品不足。
- **二次确认**：点击后先展示确认信息，包含：
  - 当前坐骑名称和品阶
  - 当前进阶祝福值和进阶祝福总值
  - 运气最好时需要的进阶物品数量（假设每次获得最大祝福值）
  - 运气最差时需要的进阶物品数量（假设每次获得最小祝福值）
  - 玩家当前拥有的进阶物品数量
  - 确认按钮和取消按钮
- 确认后逐个消耗进阶物品，每个消耗后进行一次进阶判定。
- 如果中途进阶成功，停止消耗，提示进阶成功。
- 如果全部消耗完毕仍未进阶，提示当前祝福值和剩余状态。

运气最差和最好计算方式：

- **运气最好**：`ceil(剩余祝福缺口 / 每次最大祝福值)`，即每次都获得最大祝福值。
- **运气最差**：`ceil(剩余祝福缺口 / 每次最小祝福值)`，即每次都获得最小祝福值。
- 如果当前已有祝福值，剩余祝福缺口 = 进阶祝福总值 - 当前进阶祝福值。
- 注意：运气最差不代表一定能进阶，只是保证祝福值填满的最少次数；填满后概率为 100%。

## 六、显化方向选择流程

十阶玉麒麟满星进阶成功后触发选择流程：

### 6.1 首次触发

1. 系统提示：「玉麒麟满星进阶成功！请选择显化方向：」
2. 弹出四个按钮，每个按钮展示方向名称、姿态名称和大道显化说明：

```text
**选择显化方向**

[东方·青龙 苍龙吟霄 · 生机勃发，春雷化雨]
[西方·白虎 庚金啸风 · 肃杀凛冽，秋霜断魂]
[南方·朱雀 离火焚穹 · 涅槃重生，夏阳炽烈]
[北方·玄武 玄冥镇渊 · 不动如山，冬水封天]
```

3. 玩家点击按钮后，坐骑变为对应方向的 0 星显化坐骑，`manifest_chosen` 设为 1。
4. 选择不可更改。

### 6.2 未选择时的拦截逻辑

如果玩家在十阶满星进阶成功后未选择显化方向（未点击任何方向按钮），则：

- **每次使用坐骑相关命令时**（坐骑、坐骑升星、坐骑进阶、一键升星、一键进阶、坐骑帮助），都会被拦截并重新弹出显化方向选择界面。
- 选择界面展示四个方向、对应的坐骑名称、对应的大道显化说明。
- 玩家点击选择后才能继续正常使用坐骑命令。
- 帮助命令除外：坐骑帮助不拦截，正常展示帮助信息。

```sql

    manifest_aphorism TEXT NOT NULL DEFAULT '',

    created_at TEXT NOT NULL DEFAULT ''
);
```

字段说明：

| 字段 | 说明 |
|---|---|
| mount_id | 稳定 ID，如 `qj_1`、`my_2`、`manifest_east`、`extreme_east` |
| tier | 品阶序号，1-10 对应一阶至十阶；显化 11-14；极显化 21-24 |
| name | 坐骑名称 |
| lore | 意境说明 / 大道显化 / 极境说明 |
| initial_stars | 初始星数，一般为 0 |
| max_stars | 满星数，普通和显化为 10；极显化为 114514 |

| advance_blessing_min | 每次进阶物品增加的祝福值下限 |
| advance_blessing_max | 每次进阶物品增加的祝福值上限 |
| advance_probability_formula | 进阶概率公式描述 |
| star_item_id | 升星物品 ID |
| advance_item_id | 进阶物品 ID，极显化为空字符串 |
| mount_type | 类型：`normal` / `manifest` / `extreme` |
| manifest_direction | 显化方向名称（仅显化和极显化有值） |
| manifest_aphorism | 大道显化或极境说明（仅显化和极显化有值） |
| post_advance_mount_id | 进阶后的坐骑 ID（仅普通和显化有值） |

### 2. 玩家坐骑表

表名：`player_mounts`

```sql
CREATE TABLE IF NOT EXISTS player_mounts (
    client_id TEXT PRIMARY KEY,
    mount_id TEXT NOT NULL,
    stars INTEGER NOT NULL DEFAULT 0,
    blessing_value INTEGER NOT NULL DEFAULT 0,
    blessing_expires_at TEXT NOT NULL DEFAULT '',
    manifest_chosen INTEGER NOT NULL DEFAULT 0,
    created_at TEXT NOT NULL,
    updated_at TEXT NOT NULL
);
```

字段说明：

| 字段 | 说明 |
|---|---|
| client_id | 玩家 ID，主键，每名玩家仅有一个坐骑 |
| mount_id | 当前坐骑 ID，关联 mount_defs.mount_id |
| stars | 当前星数 |
| blessing_value | 当前进阶祝福值 |
| blessing_expires_at | 进阶祝福值过期时间，ISO 8601 格式；空字符串表示无计时 |
| manifest_chosen | 是否已选择显化方向（0 未选择，1 已选择）；当 mount_type = 'normal' 且 tier = 10 且满星进阶成功时设为 0，选择后设为 1 |
| created_at | 创建时间 |
| updated_at | 最后更新时间 |

说明：

- 每名玩家有且仅有一个坐骑记录。
- 坐骑进阶时 mount_id 更新为下一品阶的 mount_id，stars 归零。
- 显化选择时 mount_id 更新为对应方向的 manifest mount_id，stars 归零，manifest_chosen 设为 1。

## 十一、开通条件

- 玩家等级 ≥ 10 时自动开通坐骑模块。
- 开通时自动创建 `player_mounts` 记录，mount_id = `qj_1`，stars = 0。
- 等级不足 10 时使用坐骑相关命令提示「等级达到 10 级后开启坐骑模块」。
