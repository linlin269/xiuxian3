# 网页链接格式修改方案

## 背景

当前 `PROJECT_DOMAIN=https://5eb98e9d.r23.cpolar.top`，已经通过 cpolar 映射到本地 `127.0.0.1:1234`。

但代码中拼接外部链接时，会自动追加 `SERVER_PORT`（1234），导致生成的链接是：
```
https://5eb98e9d.r23.cpolar.top:1234/xiuxian/help
https://5eb98e9d.r23.cpolar.top:1234/xiuxian/zhandou-rizhi/explore/1
```

期望格式：
```
https://5eb98e9d.r23.cpolar.top/xiuxian/help
https://5eb98e9d.r23.cpolar.top/xiuxian/zhandou-rizhi/explore/1
```

## 受影响文件

### 1. `xiuxian3/xiuxianserver/修仙/battle_log_links.py`

**函数**: `_base_url()`（第29-48行）

**当前逻辑**:
```python
port = str(config.server.port)
domain = (config.project.domain or "127.0.0.1").strip().rstrip("/")
# ... 解析 scheme/host/path ...
final_port = explicit_port or port  # ← 问题在这里：没有显式端口时用 server.port 补上
netloc = hostname if final_port == "80" else f"{hostname}:{final_port}"
```

**修复逻辑**:
- 如果 `config.project.domain` 已配置（非空），直接用它作为 base URL，不追加端口
- 只有 `config.project.domain` 为空时，才回退到 `http://127.0.0.1:{port}`

```python
def _base_url() -> str:
    domain = (config.project.domain or "").strip().rstrip("/")
    if domain:
        return domain  # 已配置域名，直接使用（如 https://5eb98e9d.r23.cpolar.top）
    # 未配置域名，回退到本地地址
    port = str(config.server.port)
    return f"http://127.0.0.1:{port}"
```

### 2. `xiuxian3/xiuxianserver/修仙/修仙帮助/service.py`

**函数**: `_help_page_url()`（第16-22行）和 `_with_project_port()`（第25-42行）

**当前逻辑**:
```python
port = str(config.server.port)
domain = (config.project.domain or "127.0.0.1").strip().rstrip("/")
base_url = _with_project_port(domain, port)  # ← 自动追加端口
return f"{base_url}/xiuxian/help"
```

**修复逻辑**: 与 `battle_log_links.py` 相同

```python
def _help_page_url() -> str:
    domain = (config.project.domain or "").strip().rstrip("/")
    if domain:
        base_url = domain
    else:
        port = str(config.server.port)
        base_url = f"http://127.0.0.1:{port}"
    return f"{base_url}/xiuxian/help"
```

`_with_project_port()` 函数可以保留（不影响功能），但 `_help_page_url()` 不再调用它。

## 不需要修改的地方

- **服务端路由**：`@router.get("/xiuxian/help")` 等是相对路径路由，自动适配访问域名
- **HTML 内部链接**：`<a href="/xiuxian/help">` 是相对路径，不涉及域名拼接
- **bot 端 WebSocket**：`ws://127.0.0.1:1234/ws/bot` 是内部通信地址
- **冒烟测试**：只检查路径片段（如 `zhandou-rizhi/explore`），不涉及完整域名

## 本地访问不受影响

服务端仍然监听 `0.0.0.0:1234`，所以 `http://127.0.0.1:1234/xiuxian/help` 仍然可以正常访问。修改的只是"发给用户的链接格式"。
