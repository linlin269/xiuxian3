# 后台接口组件说明

后台接口组件是 web 后台 API 的路由承接点，负责承接管理端 HTTP 页面与操作接口。

## HTTP

| 路径 | 说明 |
|------|------|
| `GET /xiuxian/admin` | 后台首页 |
| `GET /xiuxian/admin/login` | 后台登录页 |
| `GET /xiuxian/admin/operations` | 后台操作页 |
| `POST /xiuxian/admin/api/hall-of-heroes/refresh` | 手动刷新英灵殿 NPC |

## 规则

- 后台接口只承接 HTTP API，不挂载 WS 玩家命令。
- 玩家玩法仍以 WS 命令为主，不能为了后台查询绕过业务服务里的校验和结算规则。
- 后台页面上的按钮必须对应明确的后端动作，写操作接口要先校验管理员会话。
- 后台接口只服务管理和观测，不直接替代玩家领取、出售、战斗、宗门战等正式入口。

## 组件关联

- 根模块会把 `后台接口.router` include 到修仙总 router；没有 router 的二级组件不会进入 HTTP 路由。
- 后台接口可以读取修仙公共数据库和公共服务，但不能反向导入其它二级组件。
- 后台接口展示玩家、物品、世界状态、宗门、战斗日志或备份信息时，必须同步更新对应组件说明，避免后台口径和玩家口径分叉。
- 如果后台按钮需要调用某个玩法模块的写操作，优先复用该模块公开的服务方法，再由后台路由做权限和返回值包装。

## 扩展

- 新增后台页面数据接口时，优先在这里扩展路由。
- 涉及业务写操作时，应复用根目录公共能力或对应服务的公共抽象，不复制结算公式。
